# Несправлятор

Telegram-бот для групповых чатов. Отслеживает репутацию несправления участников через систему баллов: пользователи могут начислять друг другу плюсы и минусы, делать ставки и следить за статистикой. Дополнительно умеет уведомлять о событиях в голосовых каналах Discord.

## Возможности

- **Система баллов** — ответь на сообщение с `+N` или `-N`, чтобы начислить или списать баллы. Размер ограничен только твоим дневным лимитом раздачи (потолка за одну операцию нет)
- **Дневной лимит** — каждый участник может раздать `DAILY_LIMIT` баллов в сутки (по умолчанию 1000), лимит сбрасывается в полночь UTC. Может накапливаться выше базового за счёт выигрышей в `/bet` и админских начислений
- **Массовое начисление** — ответь на сообщение бота с баллами, и они распределятся на всех участников чата
- **Ставки** — `/bet`, ежедневный бросок кубика: выпало 4–6 — `+1000` к дневному лимиту раздачи (накапливается), выпало 1–3 — `−1000` к своим баллам
- **Статистика** — таблица лидеров и личная статистика по командам `/stats` и `/me`
- **Discord-интеграция** — привяжи Discord-сервер к чату и получай уведомления, когда кто-то заходит или выходит из голосового канала

### Команды

| Команда                      | Описание                                         |
| ---------------------------- | ------------------------------------------------ |
| `/start`                     | Приветствие                                      |
| `/help`                      | Справка по боту                                  |
| `/me`                        | Личный баланс и остаток дневного лимита          |
| `/stats`                     | Таблица лидеров                                  |
| `/balances`                  | Остатки дневных лимитов всех участников          |
| `/bet`                       | Бросить кубик                                    |
| `/add @user +/-N`            | Изменить лимит пользователя (только супер-админ) |
| `/discord_bind <guild_id>`   | Привязать Discord-сервер                         |
| `/discord_unbind <guild_id>` | Отвязать Discord-сервер                          |

## Стек

- **Go 1.25**
- **SQLite** — pure-Go драйвер [modernc.org/sqlite](https://pkg.go.dev/modernc.org/sqlite) (без cgo)
- **[sqlc](https://sqlc.dev)** — типобезопасный доступ к БД, код генерируется из SQL
- **[goose](https://github.com/pressly/goose)** — миграции БД (встроены и применяются на старте)
- **[Task](https://taskfile.dev)** — таск-раннер
- **[telebot](https://github.com/tucnak/telebot)** — Telegram Bot API
- **[discordgo](https://github.com/bwmarrin/discordgo)** — Discord API
- **Docker**

Источник правды для баллов и лимитов — append-only таблица `postings` (ledger): баланс и дневной остаток считаются как сумма постингов. Схема для sqlc лежит в `db/schema.sql`, запросы — в `db/queries/`, миграции — в `db/migrations/`.

## Локальная разработка

Требуется **Go 1.25+** и **[Task](https://taskfile.dev/installation/)**.

```bash
# 1. поставить dev-инструменты (sqlc, goose)
task tools

# 2. сгенерировать код доступа к БД (он не коммитится в git)
task generate

# 3. настроить окружение
cp .env.example .env   # заполнить токены

# 4. запустить
task run
```

Полезные команды:

| Команда                       | Описание                                              |
| ----------------------------- | ----------------------------------------------------- |
| `task generate`               | Сгенерировать код БД из `db/queries` через sqlc       |
| `task test`                   | Прогнать тесты                                         |
| `task build`                  | Собрать бинарь в `bin/bot`                            |
| `task migrate`                | Применить миграции к `$DB_PATH` (без запуска бота)     |
| `task migrate:create -- <имя>`| Создать новую SQL-миграцию                             |
| `task` (по умолчанию)         | `fmt` + `generate` + `build`                          |

> Сгенерированный sqlc-код (`internal/database/gen/`) намеренно в `.gitignore` — после клона запусти `task generate`. Миграции применяются автоматически при старте бота; `task migrate` нужен для ручного прогона (например на копии прод-БД).

## Деплой

### 1. Подготовка

Склонируй репозиторий и создай файл `.env` на основе [.env.example](.env.example).

### 2. Запуск

```bash
docker compose up -d --build
```

Бот запустится в контейнере, база данных сохраняется в `./data/`. Docker-сборка сама генерирует sqlc-код и применяет миграции на старте.

### 3. Остановка

```bash
docker compose down
```
